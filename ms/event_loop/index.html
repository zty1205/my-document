关键步骤如下：

  1. 执行 queue 最前面的 macro-task
  2. 检查该 macro-task 是否存在 micro-task，如果有则执行
  3. 执行下一行 macro-task。 不断重复上面的过程，直至 queue 为空
  4. 执行render

macro-task 和 micro-task

  • macro-task(宏任务)主要包含：setTimeout、setInterval、setImmediate、I/O、UI交互事件、（整体代码script）
  • micro-task(微任务)主要包含：Promise、process.nextTick、Mutation Observer(监听DOM树变化)
  • macro-task 和 micro-task 都有自己的

在一个 macro-task 中

  • 同步任务进入主线程
  • 异步任务进入Event Table并注册函数
  • 当指定的事情完成时，Event Table 会将这个函数移入 Event Queue
  • 主线程内的任务执行完毕为空，会去Event Queue读取对应的函数，进入主线程执行
  • 上述过程会不断重复，也就是常说的Event Loop(事件循环)
